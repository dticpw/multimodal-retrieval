Multimodal RAG System

基于 CLIP + FAISS + LLM 的多模态检索增强生成系统

GitHub Source

项目简介

端到端多模态 RAG 系统，支持文本搜图、以图搜图、RAG 问答三种检索模式。 使用 CLIP (ViT-B/32) 进行跨模态编码，FAISS 实现高效向量检索，结合 LLM 生成自然语言回答。 在 Flickr30K 数据集上，Text-to-Image R@1 达到 60.50%，对齐公开基线水平。

功能特性

文本搜图 (Text → Image)

输入自然语言描述，检索语义最匹配的图片。CLIP 跨模态对齐实现零样本检索。

以图搜图 (Image → Image)

上传一张图片，找到视觉特征最相似的图片。基于 CLIP 视觉编码器的语义匹配。

RAG 问答

结合检索结果和 LLM，对图片知识库进行问答。检索提供证据，LLM 组织回答。

评测对齐基线

Flickr30K 标准评测：T→I R@1=60.50%, R@5=86.76%；I→T R@1=80.00%, R@5=95.70%

快速开始

# 1. 克隆仓库
git clone https://github.com/your-username/multimodal-retrieval.git
cd multimodal-retrieval

# 2. 安装依赖
pip install -r requirements.txt

# 3. 数据摄入 (需要本地 Flickr30K 数据集 + CLIP 模型)
python -m scripts.ingest

# 4. 启动 Gradio Demo
python -m app.demo
# 浏览器打开 http://localhost:7860

# 或启动 FastAPI 服务
uvicorn app.main:app --reload
# API 文档: http://localhost:8000/docs
